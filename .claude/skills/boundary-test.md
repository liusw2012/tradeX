name: boundary-test
description: 执行边界测试评估，验证系统在边界条件和极限情况下的行为是否符合预期
---

## 评估维度

### 1. 数值边界 (Numeric Boundaries)
- 极值测试：最大/最小值测试
- 零值测试：零及附近值测试
- 溢出测试：数值溢出处理
- 精度测试：浮点数精度问题

### 2. 数据边界 (Data Boundaries)
- 空值测试：NULL/空字符串测试
- 空集合测试：空数组/空列表测试
- 长度边界：最大/最小长度测试
- 容量边界：容量限制测试

### 3. 状态边界 (State Boundaries)
- 初始状态：系统初始状态测试
- 终态测试：流程结束状态测试
- 状态转换：状态转换边界测试
- 并发状态：并发边界条件测试

### 4. 时间边界 (Time Boundaries)
- 时区边界：夏令时/跨时区测试
- 时间戳边界：时间戳边界测试
- 超时测试：超时边界测试
- 日期边界：月末/年初等日期测试

### 5. 并发边界 (Concurrency Boundaries)
- 并发数量：最大并发数测试
- 资源竞争：资源竞争边界测试
- 死锁测试：死锁条件测试
- 负载极限：极限负载测试

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 边界覆盖度 | (已测试边界数 / 识别边界数) × 100% | >= 90% |
| 边界用例通过率 | (通过边界用例数 / 执行边界用例数) × 100% | 100% |
| 异常处理覆盖率 | (已处理异常边界数 / 异常边界总数) × 100% | >= 95% |
| 未预见问题数 | 发现的新边界问题数 | <= 3 |

## 评估标准

- **通过**: 边界覆盖度 >= 90% 且边界用例通过率 = 100% 且异常处理覆盖率 >= 95%
- **警告**: 边界覆盖度 >= 80% 且边界用例通过率 >= 95%
- **不通过**: 边界覆盖度 < 80% 或存在未处理的Critical边界问题

## 执行步骤

1. 识别系统中的各种边界条件
2. 收集边界测试用例执行结果
3. 分析边界覆盖情况
4. 检查异常处理
5. 评估边界问题影响
6. 生成评估报告

## 注意事项

- 重点关注核心模块的边界条件
- 检查极限情况的系统行为
- 确保异常情况正确处理
- 使用中文输出报告
