## 评估维度

### 1. 响应时间 (Response Time)
- 平均响应时间：请求的平均处理时间
- P95/P99响应时间：高性能要求下的响应时间
- 最大响应时间：最慢请求的处理时间
- 响应时间分布：响应时间的分布情况

### 2. 吞吐量 (Throughput)
- TPS：每秒事务处理数
- QPS：每秒查询数
- 并发用户数：支持的并发用户数
- 吞吐量趋势：不同负载下的吞吐量

### 3. 资源利用 (Resource Utilization)
- CPU利用率：CPU使用率是否合理
- 内存利用率：内存使用是否在合理范围
- 磁盘I/O：磁盘读写是否成为瓶颈
- 网络I/O：网络带宽是否充分利用

### 4. 稳定性 (Stability)
- 长时间运行：是否支持24小时以上运行
- 内存泄漏：是否存在内存泄漏
- 资源回收：资源是否正常释放
- 错误率：长时间运行的错误率

### 5. 性能瓶颈 (Bottleneck)
- 瓶颈识别：是否识别出性能瓶颈
- 瓶颈定位：瓶颈是否精确定位
- 优化建议：是否有优化建议

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 平均响应时间 | 统计平均值 | < 100ms |
| P99响应时间 | 99分位值 | < 500ms |
| TPS | 每秒事务数 | >= 1000 |
| 并发用户数 | 最大并发用户 | >= 100 |
| 错误率 | (错误请求数 / 总请求数) × 100% | < 0.1% |
| CPU利用率 | 峰值CPU使用率 | < 80% |
| 内存利用率 | 峰值内存使用率 | < 85% |

## 评估标准

- **通过**: 所有指标达到阈值要求
- **警告**: 关键指标接近阈值
- **不通过**: 关键指标超过阈值或存在内存泄漏

## 执行步骤

1. 收集性能测试报告数据
2. 分析各项性能指标
3. 检查资源利用情况
4. 评估系统稳定性
5. 识别性能瓶颈
6. 生成评估报告

## 注意事项

- 确保测试环境与生产环境一致
- 重点关注响应时间和吞吐量
- 检查长时间运行的稳定性
- 使用中文输出报告
