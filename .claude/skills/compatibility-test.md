## 评估维度

### 1. 操作系统兼容性 (OS Compatibility)
- Windows版本：是否支持各Windows版本
- Linux发行版：是否支持主流Linux发行版
- macOS版本：是否支持各macOS版本

### 2. 浏览器兼容性 (Browser Compatibility)
- 主流浏览器：Chrome/Firefox/Edge/Safari
- 浏览器版本：各浏览器的多个版本
- JavaScript引擎：各浏览器的JS执行

### 3. 硬件兼容性 (Hardware Compatibility)
- CPU架构：x86/ARM等
- 内存要求：最低/推荐内存
- 磁盘空间：最低磁盘空间要求

### 4. 依赖兼容性 (Dependency Compatibility)
- 数据库版本：支持的数据库版本
- 中间件版本：支持的中间件版本
- SDK版本：各语言SDK版本

### 5. 数据兼容性 (Data Compatibility)
- 数据迁移：旧版本数据能否迁移
- 协议兼容：通信协议是否兼容
- API兼容：API版本兼容性

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| OS兼容性 | (兼容OS数 / 测试OS数) × 100% | >= 90% |
| 浏览器兼容性 | (兼容浏览器数 / 测试浏览器数) × 100% | >= 85% |
| 硬件兼容性 | (兼容配置数 / 测试配置数) × 100% | 100% |
| 依赖兼容性 | (兼容版本数 / 测试版本数) × 100% | >= 90% |
| 数据兼容性 | 迁移成功率 | 100% |

## 评估标准

- **通过**: OS兼容性 >= 90% 且依赖兼容性 >= 90% 且数据兼容性 = 100%
- **警告**: OS兼容性 >= 80% 且依赖兼容性 >= 80%
- **不通过**: OS兼容性 < 80% 或存在关键功能不兼容

1. 收集兼容性测试报告
2. 分析各维度的兼容性结果

4. 评估影响范围
5. 生成评估报告
