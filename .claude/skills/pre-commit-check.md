---

## 评估维度

### 1. 代码格式检查 (Format)
- 缩进格式：是否符合项目规范
- 行尾字符：是否统一(CRLF/LF)
- 行尾空格：是否清理多余空格
- 行长度：是否超过最大长度限制

### 2. 静态分析 (Static Analysis)
- 编译检查：代码是否能编译通过
- 静态分析工具：是否有警告
- 依赖检查：依赖是否完整

### 3. 语法和风格 (Syntax & Style)
- Google C++ 规范：是否符合编码规范
- 命名规范：变量/函数命名是否规范
- 注释规范：注释是否完整

### 4. 差量检查 (Diff)
- 新增代码：是否有充分的测试
- 修改范围：修改是否过于庞大
- 敏感信息：是否包含密钥/密码

### 5. Git钩子检查 (Git Hooks)
- commit-msg：提交信息是否符合规范
- pre-commit：是否执行必要的检查
- 差量大小：提交差量是否合理

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 格式合规率 | (合规文件数 / 检查文件数) × 100% | >= 95% |
| 静态分析通过率 | (通过文件数 / 检查文件数) × 100% | >= 90% |
| 敏感信息检查 | 扫描结果 | 0 泄漏 |
| 提交信息规范率 | (规范提交数 / 总提交数) × 100% | 100% |

## 评估标准

- **通过**: 格式合规率 >= 95% 且静态分析通过率 >= 90% 且无敏感信息泄漏
- **警告**: 格式合规率 >= 85% 且静态分析通过率 >= 80%
- **不通过**: 格式合规率 < 85% 或存在敏感信息泄漏或编译失败

```

- 检查时间: YYYY-MM-DD

- 检查结果: [通过/警告/不通过]

## 详细检查结果

### 1. 代码格式检查
- 结果: [通过/警告/失败]
- 问题文件: [列表]

### 2. 静态分析检查
- 结果: [通过/警告/失败]
- 警告/错误: [列表]

### 3. Git钩子检查
- 结果: [通过/警告/失败]
- 问题: [列表]

### 4. 敏感信息检查
- 结果: [通过/失败]
- [检查结果详情]

## 问题汇总
- [问题列表及修复建议]

[通过/警告/不通过] - 详细说明
```

## 执行步骤

1. 获取当前Git差量( staged changes )
2. 对差量文件进行格式检查
3. 运行静态分析工具
4. 检查敏感信息泄露
5. 验证提交信息格式
6. 生成检查报告

## 注意事项

- 建议配置Git hooks实现自动化
- 检查应快速执行(<30秒)
- 关键问题应阻止提交
- 使用中文输出报告
