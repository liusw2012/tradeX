---

## 评估维度

### 1. 预案覆盖 (Coverage)
- 故障场景：是否覆盖常见故障场景
- 业务影响：是否覆盖重要业务
- 恢复场景：是否覆盖各种恢复场景

### 2. 预案质量 (Quality)
- 步骤清晰：预案步骤是否清晰
- 责任明确：责任人是否明确
- 时限明确：处理时限是否明确
- 验证机制：是否有验证机制

### 3. 演练验证 (Drill)
- 演练频率：是否定期演练
- 演练记录：演练是否有记录
- 问题改进：演练问题是否改进

### 4. 响应流程 (Response)
- 响应速度：响应是否及时
- 升级机制：是否明确升级机制
- 沟通机制：沟通机制是否顺畅

### 5. 资源准备 (Resources)
- 工具准备：应急工具是否准备
- 权限准备：应急权限是否准备
- 备件准备：备件是否准备

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 预案覆盖率 | (已有预案数 / 应有预案数) × 100% | >= 90% |
| 预案完整性 | 预案内容完整性评分 | >= 85% |
| 演练覆盖率 | (已演练预案数 / 应演练预案数) × 100% | >= 80% |
| 响应时间 | 从故障发现到响应的平均时间 | < 5分钟 |
| 恢复时间 | 从故障发生到恢复的平均时间 | < 30分钟 |

## 评估标准

- **通过**: 预案覆盖率 >= 90% 且预案完整性 >= 85% 且演练覆盖率 >= 80%
- **警告**: 预案覆盖率 >= 80% 且预案完整性 >= 75%
- **不通过**: 预案覆盖率 < 80% 或存在关键业务无预案

## 执行步骤

1. 检查现有应急预案
2. 评估预案覆盖范围
3. 检查预案质量和完整性
4. 检查演练记录
5. 评估响应流程
6. 生成评估报告

## 注意事项

- 重点关注核心业务的应急预案
- 确保预案步骤可执行
- 检查演练的有效性
- 使用中文输出报告
