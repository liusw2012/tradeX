## 评估维度

### 1. 发布策略 (Release Strategy)
- 发布方式：灰度/蓝绿/滚动发布
- 发布粒度：功能/模块发布粒度
- 回滚策略：回滚方案是否完善
- 验证策略：发布后验证方案

### 2. 风险评估 (Risk Assessment)
- 风险识别：是否识别所有风险
- 风险等级：风险等级是否合理
- 缓解措施：是否有风险缓解措施
- 应急预案：是否有应急预案

### 3. 资源规划 (Resource Planning)
- 人员安排：发布人员是否到位
- 资源准备：服务器/网络是否准备好
- 时间安排：发布时间窗口是否合适
- 职责分工：职责是否清晰

### 4. 变更管理 (Change Management)
- 变更审批：是否有审批流程
- 变更记录：变更是否记录完整
- 沟通计划：是否制定沟通计划
- 通知机制：是否通知相关方

### 5. 监控计划 (Monitoring Plan)
- 监控指标：监控指标是否明确
- 告警阈值：告警阈值是否设置
- 巡检计划：是否有巡检计划
- 应急响应：应急响应流程是否清晰

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 风险覆盖率 | (已识别风险项 / 应识别风险项) × 100% | >= 90% |
| 缓解措施完备率 | (有缓解措施的风险 / 总风险) × 100% | 100% |
| 回滚方案完备率 | 回滚方案完整性评分 | >= 90% |
| 监控指标覆盖率 | (有监控的指标数 / 应监控指标数) × 100% | >= 95% |
| 人员就绪率 | (就绪人员数 / 应需人员数) × 100% | 100% |

## 评估标准

- **通过**: 风险覆盖率 >= 90% 且缓解措施完备率 = 100% 且回滚方案完备率 >= 90%
- **警告**: 风险覆盖率 >= 80% 且缓解措施完备率 >= 90%
- **不通过**: 风险覆盖率 < 80% 或缺少关键回滚方案

## 执行步骤

1. 读取发布方案文档
2. 评估发布策略的合理性
3. 识别并评估风险
4. 检查回滚方案
5. 评估监控计划
6. 生成评审报告

## 注意事项

- 重点关注回滚方案的可行性
- 检查风险缓解措施的有效性
- 确保监控指标覆盖关键业务
- 使用中文输出报告
