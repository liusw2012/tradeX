## 评估维度

### 1. 目标达成 (Goal Achievement)
- 功能目标：功能是否按计划完成
- 性能目标：性能是否达到预期
- 时间目标：是否按时完成
- 资源目标：资源使用是否合理

### 2. 质量评估 (Quality Assessment)
- 缺陷数量：发布后发现的问题数
- 缺陷严重度：缺陷的严重程度分布
- 回滚次数：是否需要回滚
- 热修复次数：是否需要热修复

### 3. 风险控制 (Risk Control)
- 风险发生：识别的风险是否发生
- 新风险发现：是否有新风险
- 风险影响：风险造成的影响
- 风险处理：风险处理是否得当

### 4. 经验总结 (Lessons Learned)
- 成功经验：成功的做法
- 失败教训：失败的教训
- 改进建议：后续改进建议

### 5. 相关方满意度 (Stakeholder Satisfaction)
- 用户反馈：用户是否满意
- 团队反馈：团队是否满意
- 业务方反馈：业务方是否满意

## 量化指标

| 指标 | 计算方式 | 通过阈值 |
|------|----------|----------|
| 目标达成率 | (达成目标数 / 计划目标数) × 100% | >= 95% |
| 缺陷密度 | 缺陷数 / KLoC | < 1.0 |
| 回滚率 | (回滚次数 / 总发布数) × 100% | < 2% |
| 目标偏差 | (实际值 - 预期值) / 预期值 × 100% | < 10% |
| 满意度评分 | 满意度调查评分 | >= 4.0/5.0 |

## 评估标准

- **通过**: 目标达成率 >= 95% 且缺陷密度 < 1.0 且回滚率 < 2%
- **警告**: 目标达成率 >= 90% 且缺陷密度 < 1.5
- **不通过**: 目标达成率 < 90% 或存在Critical缺陷或回滚率 >= 2%

## 执行步骤

1. 收集发布执行数据
2. 对比目标与实际结果
3. 分析缺陷情况
4. 评估风险控制
5. 总结经验教训
6. 生成评估报告

## 注意事项

- 客观评估目标达成情况
- 重点关注缺陷分析
- 总结可复用的经验
- 使用中文输出报告
