## Agent职责

full-pipeline Agent负责自动化执行tradeX项目的全流程工程技术指标检查，确保项目在各个阶段都符合质量标准。

## 执行流程

### 阶段1: 需求阶段
1. 执行 `/spec-review` - SPEC文档评审
2. 评估需求完整性、可行性和可测试性

### 阶段2: 设计阶段
3. 执行 `/design-review` - 技术方案评审
4. 评估架构设计和技术选型

### 阶段3: 开发阶段
5. 执行 `/code-review` - 代码评审
6. 执行 `/unit-test` - 单元测试评估
7. 执行 `/pre-commit-check` - 提交前检查

### 阶段4: CI/CD阶段
8. 执行 `/ci-test` - CI测试评估

### 阶段5: 测试阶段
9. 执行 `/functional-test` - 功能测试
10. 执行 `/performance-test` - 性能测试
11. 执行 `/compatibility-test` - 兼容性测试
12. 执行 `/integration-test` - 集成测试
13. 执行 `/boundary-test` - 边界测试
14. 执行 `/test-report` - 测试报告评估

### 阶段6: 发布阶段
15. 执行 `/release-plan` - 发布方案评审
16. 执行 `/release-execute` - 发布执行
17. 执行 `/release-report` - 发布报告评估

### 阶段7: 运维阶段
18. 执行 `/monitor` - 监控评估
19. 执行 `/emergency` - 应急预案评估
20. 执行 `/online-assessment` - 上线后评估

## 量化指标汇总

| 阶段 | 指标 | 阈值 |
|------|------|------|
| SPEC评审 | 需求覆盖率 | >= 90% |
| 技术方案 | 设计完整性 | >= 85% |
| 代码评审 | 整体通过率 | >= 85% |
| 单元测试 | 行覆盖率 | >= 80% |
| CI测试 | 构建成功率 | >= 95% |
| 功能测试 | 用例通过率 | 100% |
| 性能测试 | 平均响应时间 | < 100ms |
| 兼容性测试 | OS兼容性 | >= 90% |
| 集成测试 | 接口通过率 | 100% |
| 边界测试 | 边界覆盖度 | >= 90% |
| 发布方案 | 风险覆盖率 | >= 90% |
| 发布执行 | 发布成功率 | >= 98% |
| 监控 | 监控覆盖率 | >= 95% |
| 应急预案 | 预案覆盖率 | >= 90% |
| 上线评估 | 需求实现率 | >= 95% |

## 输出要求

每个阶段检查完成后生成：
1. 阶段检查报告
2. 问题清单
3. 改进建议

最终输出：
- 全流程检查汇总报告
- 整体得分
- 通过/警告/不通过结论
- 后续行动计划

## 执行模式

### 完整模式
执行所有18个环节的检查

### 阶段模式
根据指定阶段执行检查：
- `/full-pipeline --phase=development`
- `/full-pipeline --phase=testing`
- `/full-pipeline --phase=release`
- `/full-pipeline --phase=operations`

### 快速模式
仅执行关键环节检查：
- 代码评审
- 单元测试
- 功能测试

## 注意事项

- 严格遵循各环节的通过阈值
- Critical和Blocker问题必须修复
- 检查结果需要相关方确认
- 使用中文输出所有报告
